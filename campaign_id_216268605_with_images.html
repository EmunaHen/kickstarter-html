Story

Introducing CinematicNode

A visual automation studio for creators who want to make smarter videos—without timelines or code.

CinematicNode is a next-generation desktop tool that lets you build videos using a powerful visual node system. Whether you're generating Shorts, automating overlays, adding subtitles, or integrating AI narration, CinematicNode helps you create content faster and smarter.

Think of it like a creative command center—each node represents a function, like text generation, audio effects, or stock video search. By connecting them together, you design a video flow that automates everything from start to render.

Why We're Building This

As developers and creators, we were frustrated with how slow, clunky, and repetitive most video editing workflows are. We wanted something that felt like a mix between a video editor, an automation tool, and a creative playground.

So we started prototyping CinematicNode. What began as a tool for ourselves quickly turned into something we realized others would want too—especially creators working with AI, storytelling, social content, or automation pipelines.

The Problem We’re Solving

AI tools have made it easier than ever to generate audio, video, and text—but consistency is still a huge problem. Most creators are stitching tools together manually, experimenting constantly, and getting different results every time. There's no clear workflow—just a patchwork of prompts, renders, and hope.

CinematicNode changes that.

It brings structure to creativity by letting you design repeatable, visual workflows that combine powerful tools—like AI narration, video overlays, summaries, stock footage, and audio effects—into a system that works the same way, every time.

You’re not just clicking “generate”—you’re designing a creative pipeline. That means faster output, fewer mistakes, and better results for things like:

  • Automated Shorts production
  • Branded storytelling formats
  • AI-assisted video creation
  • Educational and marketing content
  • Social media pipelines

Our goal is to make creativity scalable without sacrificing quality or originality.

The Plan

1. Input & Extraction:

  • File Loader
    Loads the base MP4 file.
  • Extract Audio from Video
    Separates the original MP3 track for further processing.
  • Extract Script with Timestamps
    Pulls out spoken dialogue + timestamps from the original video for subtitle and pause alignment.

2. AI Text Generation:

  • String Node 
    eg "Your given a timestamp script, I need you to convert it into the following format. The script should not change."
    Supplies a prompt or setup for the LLM.
  • String Combiner
    Combines the custom prompt with another source (likely user-defined or video metadata).
  • LLM Node
    Uses ChatGPT/GPT4 to produce a new response — either narration or commentary.

3. Voice & Subtitle Generation:

  • OpenAI Voice Generator
    Takes the LLM output and generates AI speech.
  • MP3 to WAV Converter
    Converts the MP3 to WAV format for better sync with Rhubarb.
  • Rhubarb Lip Sync
    Generates .JSON mouth-cue timing data for the avatar renderer.
  • OpenAI SRT Generator
    Creates SRT subtitle files from the audio — optionally fed into overlay node.

4. Rendering Visual Layers:

  • Avatar Renderer
    Renders a character (e.g. talking avatar) with the lip sync and background color.
  • Color Node
    Defines background color for avatar video (#FF18FF00).

5. Timing & Mutes:

  • Video Pauses Extractor
    Finds natural pauses in the original audio based on LLM cue.
  • Video Mutes Extractor
    Finds where to mute original audio based on LLM cue or silence.
  • Video Muter
    Applies mutes at defined timestamps.
  • Insert Pauses into Video
    Adds delays where needed (usually for syncing voiceover or pacing AI narration).

6. Final Assembly:

  • Object Modifier
    Adjusts the position/animation settings for overlays.
  • Overlay Video on Video
    Places the avatar or other layers onto the background video.
  • Combine Video, Audio, ASS
    Final composition step — merges video with MP3 voice and ASS (subtitle overlay).
  • Save File
    Outputs the final rendered MP4 to disk with filename set in the UI.

The Plan

1. Input & Prompt Setup

  • String
    Provides a custom text input like a Reddit story or user prompt.
    E.g., "Write a fictional scary story in the style of a Reddit user posting on r/nosleep. The story should feel personal and realistic, written in first person, and gradually build suspense. Do not include a title, subreddit reference, TLDR, or any comment-style text—just the story itself. Keep the tone casual but increasingly unsettling. Include eerie or unexplained events, and end with a creepy or ambiguous twist. The story should be around 500–700 words."
  • LLM (Top Left)
    Converts the prompt into a formatted story, script, or summary using ChatGPT.
    Output: Response text → sent to voice and caption generator

2. Voice & Caption Generation

  • OpenAI Voice Generator
    Converts the LLM response into an MP3 audio voiceover.
  • OpenAI SRT Generator
    Converts the MP3 into subtitle-format SRT based on the voice timing.
  • MP3 Info
    Extracts duration, bitrate, and file size of the audio — used for syncing or overlay calculations.

3. Video Content Automation

  • Time Video Keywords Generator
    Takes the SRT file and pulls out timestamped keywords to guide video searches.
    (Great for aligning visuals to voice pacing.)
  • LLM (Right)
    Optional second LLM pass to enhance or reformat keyword prompt.
    Outputs timestamped keywords or refined search phrases.
  • Pexels Search Settings
    Allows customization of the video search:
  • Pexels Downloader
    Fetches relevant stock clips based on the keywords + settings.
    Returns MP4 files

4. Video Assembly

  • Merge Videos
    Combines multiple stock clips into a single background MP4.
  • ASS Generator
    Converts the SRT captions into a styled ASS subtitle file.
    Takes style from the ASS Style node.

5. Final Composition

  • Combine Video, Audio, ASS
    Merges:
  • The merged Pexels MP4
  • The MP3 voiceover
  • The ASS subtitles
    into a single synchronized MP4 output.
  • Save File
    Writes the final rendered video to a selected folder with a user-defined name.

Risks and challenges

Like any software project, CinematicNode carries some development risks. The biggest challenges include ensuring cross-platform stability, refining the UI/UX for both beginner and advanced users, and meeting the delivery timeline. Our team has already built a working prototype and has experience shipping production-grade software. However, unexpected bugs, integration issues with third-party services (such as AI providers), or feedback-based changes could delay release. To mitigate this, we’ve scoped the initial launch to core features with a clear roadmap and fallback milestones. We’ll also maintain transparency with backers through regular updates and early-access builds. Your support will help us polish, test, and deliver the best experience possible.

Learn about accountability on Kickstarter

Use of AI

Our project is not raising funds for AI tools, nor are we using content that was generated by AI. However, we will be using AI by integrating existing AI services (like OpenAI or similar APIs) to automate tasks such as voice generation, summarization, or content structuring. These AI features are used as optional tools within the CinematicNode software to help creators accelerate their workflow.
