Story

What is It?

Oorian is Java-based framework for creating dynamic, interactive, data-driven web applications purely in Java allowing you to leverage all the benefits of object-oriented design in your applications front-end to back-end.  Your entire web application, including the UI, can be written in Java without the need for maintaining separate HTML, CSS, and/or JavaScript code.  As someone who is proficient in Java and highly productive when using it,  I've found trying to develop large web application UI's through a combination of HTML, CSS, and JavaScript to be incredibly tedious and inefficient at best.  Not able to find a Java-based web framework that met my needs, I created Oorian.  

Development began on Oorian several years ago to support the development of iGradePlus - a very large, data-driven web application.  Oorian has been in production use for at least 8 years as the basis for iGradePlus which now consists of hundreds of unique web pages and over 500k lines code.

Since Oorian has matured and proved to be a reliable and practical framework for developing web applications, I have decided I'd like to make it publicly available for other developers who may appreciate its benefits as much as I do.   However, because iGradePlus requires all of my time, I have not been able to get Oorian polished up enough to make it available for general use.  I am trying to raise money to hire a developer who can clean up the code and produce the documentation necessary to make it available for general use.

How Does It Work?

Oorian web applications are built and deployed just like normal Java web applications, except they include the Oorian library which provides the UI components and the translation engine for converting the pure Java UI into the HTML, CSS, and Javascript that is served to the client's browser.

Why Use It?

If you are like me and are a proficient object-oriented, Java programmer, but find web application development messy and cumbersome and just flat out despise JavaScript and the all the frameworks built for it, then you might just love Oorian.  

A Familiar Programming Model

The biggest problem I encountered when I first started developing web applications was trying to adjust to a programming model that was highly inferior to the programming model I was used to when using Java.  I think in terms of object-oriented design and in order to be productive, I need a programming language that supports that.   Oorian fills that need by allowing you to develop your entire web UI in Java using object-oriented design principles.

Extensibility and Reusability

For sophisticated UI's, your page code won't consist of hundreds of basic HTML elements.  You'll most likely be using more sophisticated components you have built such as menus, grids, dialogs, etc.  And that is one of the benefits of Oorian.  Unlike HTML where reusability means copying and pasting code or creating UI components in JavaScript, you can build a library of sophisticated controls that can be easily used throughout your web application leveraging the benefits of object-oriented programming such as inheritance and polymorphism.  With a control library layered on top of the Oorian Core library, your web application code will begin to resemble the code of a traditional GUI application.

Seamless Client-Server Communication

The second major benefit of Oorian is the seamless communication that occurs between your web app and the user's browser.  Any event that is generated in the user's browser can be easily handled in your web app running on the server simply by registering to receive the event.  It appears as if your server-side app is actually running on the client-side.

Flexibility

Oorian doesn't prevent you from using normal HTML, CSS, or JavaScript code in your project if you need to. 

Static HTML
Full static HTML pages can be included in your project and served to the client just like you would normally with a Java web application. Oorian also provides you with the ability to include HTML code fragments in an Oorian web page or element by using the HtmlFragment class.

HTML Templates
If you're building an interactive, dynamic website as opposed to a full-blown web application and are using a professionally designed template, you can still use Oorian to manage the site.  Oorian provides a special page class, HtmlTemplatePage that can be used to publish a page from an HTML file while still using all the features of Oorian to interact with page.

Roadmap

Polished Code and Documentation

The first version of Oorian will require minimal updates to the existing code and features and will mainly be producing the documentation necessary to use it and removing code that has become obsolete over the last several years.   

Single Page App Option

In it's current state, you cannot produce Single Page Applications (SPAs) with Oorian. The missing piece is a reliable navigation mechanism, which I have never had time to implement.  However, once this has been developed, Oorian should be able to produce SPAs pretty easily.  It already handles asynchronous web page updates seamlessly, it just cannot perform these updates based on user navigation.  Since SPAs have become very popular, this option is high priority.

Web Awesome and Syncfusion Integration

The original intent of Oorian was to create a Java-based framework for developing rich web applications.  The name itself is derived from this concept - Object-oriented Rich Internet Application Framework.  

Because there are so many JavaScript-based libraries for rich UI controls, it became obvious that integrating Oorian with these various libraries was easier than building a rich set of controls directly into Oorian.  However, because iGradePlus requires so much of my time, I only integrated the UI controls that I needed and I didn't integrate Oorian fully with any control libraries.   

If Oorian proves to be popular based on the performance of this campaign, integrating Oorian with popular UI control libraries will be the major focus post version 1. Syncfusion which has a comprehensive control library with a community edition is high on the candidate list.  Web Awesome, which is a new product by the people over at Font Awesome looks like another good candidate.  

However, I will be open to integrating with other libraries if users have suggestions.  It's actually pretty easy to create Oorian controls that wrap JavaScript UI controls, so community developers will be able to easily integrate with any control library they'd like if they want to develop it themselves and make it available to other users.
