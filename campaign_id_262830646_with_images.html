Story

Sushi Board & framework

Sushi Board is an ESP32-based programmable modular embedded system for general-purpose IoT, suitable for STEM education and real-world applications.

  • 100% based on widely available commercial modules, making it cost-effective and easy to source.
  • Powered by the Sushi firmware framework, which provides a complete environment integrating system management and MicroPython.
  • Programmable even without a PC, using a smartphone or tablet via a web interface.

With Sushi Board and Framework, you can focus on your application development without getting lost in low-level system details. Advanced users can still flash their own firmware, as the board is based on ESP32-DevKitC module.

Project workflow

Description

If you enjoy DIY electronics and coding, you've likely experimented with various boards and modules. When it’s time to build something practical and polished, connecting everything cleanly can be a challenge. Sushi Board simplifies this by helping you integrate modules into a neat, functional setup, ideal for IoT projects.

For MAKERs

  • A carrier board that connect all the essential components needed to create a complete IoT unit, suitable for various applications, minimizing the complexity of assembling different components and boards without requiring extensive soldering or wiring.
  • Only the CPU is mandatory; then you simply plug in the modules you need for your specific purposes.

For students and educational purposes

  • Sushi Board offers a modular system that can be shared across multiple projects, from simple setups to more complex, real-world systems.

An Arduino-style, high-level approach works well for beginners and hobbyists, but if you're interested in programming and aim to become a professional, you'll need to dive deeper into how things work behind the scenes. The Sushi Board documentation gives you the flexibility to decide your learning path.
  • A set of step-by-step software examples is provided, not just for basic tasks like ‘blinking a LED,’ but for full-scale, real-world applications, all based on the same core hardware connected to the Sushi Board.

Hardware features

All the ingredients that are needed for a myriad of applications:

  • ESP32-DevKitC V4
  • 16x digital GPIN 
  • 2x Relay OUT (10A 250V AC)
  • 5x digital direct GPIO, 1 x GPO , 2 x GPI (LEDs, extra UART, etc.) 
  • Physical keyboard
  • OLED Display 
  • Connectivity: Wi-Fi, Bluetooth, modem 2G/3G/4G, UART, I2C
  • Power supply modes: 5V (USB adapter or wired) ; Battery only ; 5V + backup battery (for when grid power is missing)
  • Backup battery
  • Power status and battery level acquisition 
  • Buzzer for audio alerts
  • Temperature sensor
  • Size: 90x110 mm (both carrier and TOP boards)

Sushi framework software features

Sushi firmware framework, provides a complete environment integrating system management and MicroPython. It simplifies development:

  1. Hardware management: Wi-Fi, LCD, keyboard, modem, I/O expansion, relays, power detection, battery level, buzzer.
  2. Software features: System management via web UI (network and system settings, logs, status etc.) and physical interface for on-device menus.
  3. embedded MicroPython environment: Users can run their own scripts freely, while built-in libraries provide optional system management features.
  4. Embedded high-level MicroPython libraries for rapid access to hardware and software functions (custom menus, web pages, I/O expansion, SMS, HTTP requests, etc.).

The Sushi firmware framework manages all the essential components, which, regardless of the specific application, are typically present in an IoT system:

  • physical interface management (keyboard, display, status LEDs)
  • wireless connection as client or access point
  • driver for all hardware components present on the board (io-expander, modem, etc)
  • web interface (web server and user interface)
  • modular menu management for physical interface
  • high level API for a lot of typical functions you need: http requests, SMS, GSM calls, web server extension, I2C communication... 

This software framework can be useful for students and for makers to accelerate projects by using solid, tested building blocks to manage core functions letting focus on the application development without getting lost in low-level system details.

 Advanced users can obviusly still flash their own firmware without using the embedded framework.

Detailed hardware features

SoC

  • "ESP32-DevKitC V4" module details: https://docs.espressif.com/projects/esp-dev-kits/en/latest/esp32/esp32-devkitc/user_guide.html
  • MCU: ESP32 WROVER/WROOM
  • EPROM: 4/8M
  • PSRAM: 4M (WROVER)

ONBOARD SOCKETS

To every socket on Sushi board can be connected an add-on module that work for a specific function. Except the ESP32 micro module all the other modules are optional, depending on your specific application.

  • MCU module with ESP32 [ESP32-DevKitC V4 module]
  • 16bit IoExpander module (digital inputs extension) [PCF8575 IO Expander module]
  • OLED display  [SSD1306 128X64 OLED module]
  • 2x relais output [Wemos D1 mini relais module]
  • Buzzer (for strongs alert beeps) [2 pin buzzer module]
  • Keyboard [4 buttons membrane keypad]
  • Battery [2x18650 Li Ion battery module]
  • Modem LTE [SIM7672G module, supports worldwide LTE(4G) bands]

CONNECTIVITY

  • WIFI 802.11 b/g/n:  client (the board connects to an existing network) or access point (the board create it's own wireless network)
  • Bluetooth: v4.2 BR/EDR and BLE
  • Modem: GSM[2G]/LTE[4G]. SMS, http connections, etc.
  • UART
  • I2C BUS 

POWER

  • 5V power supply + backup battery
  • Battery only
  • 5V power supply only

INPUT & OUTPUT

  • 16x digital inputs
  • 2x relais output
  • 3x digital output for diagnostic and status LED

INTEGRATED SENSORS

  • Cabled DS18B20 temperature sensor [default is wired DS18B20]
  • Battery level reading (ADC input)
  • Grid power OFF detection (digital input)

INTERFACE

  • membrane keyboard (default 4 buttons)
  • OLED display (default OLED SSD1306)

ONBOARD CONNECTORS

The connectors let plug the wiring to the system elements that are external to the Sushi board. You can solder both JST-XH2.54 or KF2EDG 2.54 connectors.

  • Power supply input (5V)
  • Backup battery signals
  • ON/OFF button
  • 5x digital direct GPIO, 1 x GPO , 2 x GPI (LEDs, extra UART, etc.) 
  • External temperature sensor
  • 16x general purposes digital inputs

GENERAL PURPOSE ONBOARD HEADERS

The general purpose (GP) headers let add other elements/interfaces by  the classic Dupont or similar connectors and wires.

  • External status LEDs
  • UART
  • Power supply (5V, 3.3V, GND)
  • I2C BUS
  • 2x 4 pin GP HUB (plug all pin together)
  • 1x GP ADC
  • 1x GP pull-up
  • 1x GP pull-down
  • 1x GP resistor

DIMENSIONS

  • 90x110 mm (both carrier and TOP boards)

SOME REAL APPLICATION

This is a Sushi board mounted in a junction box and installed inside an electrical panel. It functions as a weather data acquisition center, reading data from external sensors and sending it to the cloud via GSM. The same hardware can also serve as the central unit of an alarm system.

Why Sushi board ?

If you bought several boards online, conducted numerous experiments, and learned how to do many things regardless of your goals or skill level, there will likely come a moment when you think, "Okay, now I'd like to try building something real and useful, something that works for me or my home, and that I can show to my friends."
This is where a lot of new challenges arise: how do you put all the pieces together? You have several modules that perform different functions, and you need to connect them, maybe even place everything inside a box to use somewhere...
Many challenges will emerge. 
Sure, you could use a matrix board, connect the modules with jumper wires and solder joints, which works fine for testing or building a small prototype, but this approach is often tedious and results in something rather unsightly.
This is where the Sushi Board comes in, helping you connect everything in a while, fitting it all into a box with all the typical components of an IoT control center.

The name "Sushi board" comes from the idea that, like sushi—a traditional Japanese specialty—the Sushi board is composed of various ingredients (the modules) that can be added or customized based on your preferences.
